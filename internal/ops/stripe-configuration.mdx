## Webhook configuration

**Endpoint URL:** Retrieve from AppDev and store in Bitwarden.

**Events to listen for:**

- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.payment_succeeded`
- `invoice.payment_failed`

**Webhook secret:** Store in Bitwarden. The app validates this secret on every webhook event. If the secret rotates, AppDev must update the environment variable in Supabase directly — not via the Lovable UI.

**If webhook breaks:** Plan changes and subscription events stop syncing to the app. Symptoms: user upgrades but plan doesn't change in the app, or subscription cancels but access continues. Check Stripe webhook dashboard for failed deliveries. Contact AppDev immediately.

## Dunning and retry settings

Stripe default retry schedule: day 1, 3, 5, 7 after failed payment. Confirm active in Stripe Settings. Customer failure notification emails: confirm enabled before first paid subscriber converts.

## Customer portal configuration

Accessible from Clarely account settings.

- View billing history: enabled
- Update payment method: enabled
- Cancel subscription: enabled
- Change plan: disabled (plan changes handled in-app via Heimdall or upgrade flow)

Portal return URL: `clarely.app/settings/account`
