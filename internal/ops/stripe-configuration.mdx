## Webhook configuration

**Endpoint URL:** \[Retrieve from AppDev — store in Bitwarden\]

**Events to listen for:**

- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.payment_succeeded`
- `invoice.payment_failed`

**Webhook secret:** Store in Bitwarden. If the secret rotates, AppDev must update the environment variable in Supabase.

**If webhook breaks:** Plan changes and subscription events stop syncing to the app. Symptoms: user upgrades but plan doesn't update, or subscription cancels but access continues. Check Stripe webhook dashboard for failed deliveries. Contact AppDev immediately.

**Rotating the webhook signing secret:** Rotate in Stripe dashboard. Update in Supabase environment variables via the Supabase dashboard directly (not via Lovable). Update Bitwarden.

## Dunning and retry settings

Stripe default retry: day 1, 3, 5, 7. Confirm active in Stripe Settings. Customer failure notification emails: confirm enabled before first paid subscriber converts.

## Customer portal configuration

Accessible from Clarely account settings.

- View billing history: enabled
- Update payment method: enabled
- Cancel subscription: enabled
- Change plan: disabled (plan changes handled in-app)

Portal return URL: `clarely.app/settings/account`

Stripe Tax enabled. Confirm nexus states are correctly configured (Florida at minimum). Consult accountant for multi-state obligations as customer base grows.
