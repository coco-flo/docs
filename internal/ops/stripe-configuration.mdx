**If webhook breaks:** Plan changes and subscription events stop syncing to the app. Symptoms: user upgrades but plan doesn't change in the app, or subscription cancels but access continues. Check Stripe webhook dashboard for failed deliveries. Contact AppDev immediately.

**Webhook signing secret rotation:** Rotate if exposed. Update in Supabase environment variables (not in Lovable UI - use Supabase dashboard directly).

## Dunning and retry settings

Stripe default retry schedule: day 1, 3, 5, 7 after failed payment. Confirm these are active in Stripe Settings.

Customer failure notification emails: confirm enabled in Stripe Settings before first paid subscriber converts.
