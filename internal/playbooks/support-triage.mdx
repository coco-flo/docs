The founder group channel is the fastest path during the cohort period. Tally forms are for structured capture when a bug needs full reproduction steps.

## Triage decision tree

**App broken or not working as expected?**

- Yes: Create CLA- issue in Linear. Priority: Urgent if blocking a core workflow (can't send quote, can't sign, can't invoice). High otherwise. Acknowledge within 2 hours.

**How-to question?**

- Yes: Answer directly. Link to docs.clarely.co if covered. If not covered, answer and create a doc update task.

**Billing or account issue?**

- Yes: Handle directly. Check Stripe for subscription status. Check Heimdall for account status and plan. Resolve at founder level.

**Feature request or enhancement?**

- Yes: Log it. Create CLA- issue with Founder Feedback label. Route to Prod for roadmap consideration. Do not commit to timelines.

**Third-party service issue (Venmo, Cash App, Zelle, email)?**

- Yes: Acknowledge, clarify what's in your control, provide workarounds if available.

## Response time targets

| Situation | Target first response |
| --- | --- |
| App broken, can't work | 2 hours |
| Major workflow blocked | 4 hours |
| General question | 24 hours |
| Feature request | Acknowledge within 48 hours |

## Template responses

**Bug acknowledged, investigating:**

> "On it. I can reproduce this on my end. I'll have an update for you within \[timeframe\]."

**Bug fixed:**

> "This is fixed. Let me know if you see it again."

**Bug logged, not yet prioritized:**

> "Logged with high priority. I'll follow up once it's fixed. Thanks for the detail."

**Feature request acknowledged:**

> "Good idea. Logged. This isn't in the immediate roadmap but I want to capture it. I'll follow up if and when it's scoped."

**How-to question with docs link:**

> "\[Answer\]. More detail here: \[link\]."

**Third-party issue:**

> "That's a \[Venmo/Cash App/Zelle\] issue outside of Clarely's control. As a workaround, \[suggestion\]. If you keep having trouble with one method, consider setting up a second payment handle as backup."

**Account/billing question:**

> "Looking into this now. I'll check the account and follow up shortly."

## Linear issue creation from support

Every bug from a founder becomes a CLA- issue. Include: founder name (in description, not title), steps to reproduce, expected vs actual behavior, device and OS, Founder Feedback label, Tally form link if available.
